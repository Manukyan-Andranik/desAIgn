Task: Object Detection Module for Architectural Render Understanding
🎯 Goal

Build a service that receives an architectural exterior render image and returns:

A structured list of detected architectural objects
Bounding boxes for each object
Confidence scores
Normalized object classes (standard taxonomy)

This module is the first step of the Vision Pipeline.

📥 Input
Single image:
Format: JPG / PNG
Type: architectural exterior render (villa, house, building)
Resolution: 512px – 4K

Example:

input_image: villa_render.png
📤 Output (Required Schema)

Return a structured JSON:

{
  "image_id": "uuid",
  "objects": [
    {
      "id": "obj_1",
      "class": "roof",
      "bbox": [x1, y1, x2, y2],
      "confidence": 0.94
    },
    {
      "id": "obj_2",
      "class": "window",
      "bbox": [x1, y1, x2, y2],
      "confidence": 0.88
    }
  ]
}

🧠 Recommended Model Strategy

You MUST NOT train from scratch in MVP.

Use a hybrid detection stack:

Option A (Recommended MVP)
1. Grounding DINO (Text-guided detection)

Use prompts:

"roof, window, door, wall, tree, pool, car, grass, balcony"

Output:

bounding boxes
class labels
Option B (Fast production fallback)
YOLOv8 / YOLO11

Fine-tune on architectural dataset.

Best architecture:
Image
  ↓
Grounding DINO (primary detector)
  ↓
Optional YOLO refinement layer
  ↓
Post-processing normalization
🧹 Post-processing Layer (VERY IMPORTANT)

You must implement:

1. Class normalization

Map raw outputs → standard taxonomy

Example:

Raw Output	Normalized
"glass_window"	"window"
"roof_tile"	"roof"
"front_door"	"door"
2. Duplicate merging

Merge overlapping detections:

IoU threshold: 0.5–0.7
3. Confidence filtering
remove objects with confidence < 0.4 (configurable)
4. Hierarchy tagging (optional but recommended)
"parent": "building"

💡 Key Insight (important for architecture)

This module is NOT just “object detection”.

It is the entry point of Scene Understanding System.

If this layer is wrong → everything downstream (masking, editing, video) becomes wrong.

If this layer is good → you can build an entire AI architecture platform on top of it.
